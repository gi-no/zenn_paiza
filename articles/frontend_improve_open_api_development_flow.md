[paiza株式会社](https://www.paiza.co.jp/)でエンジニアをやっている川辺(yaruki00)です。

## 委員会活動について

- フロントエンド改善委員会の役割
- 半期ごとの活動テーマの決め方
- 今回の記事で扱う範囲

## 半期の目標

- OpenAPIを用いた開発フローの改善
- このテーマに取り組むことになった背景
- 改善によって目指した状態

## 改善前の課題

Issue出しのブレストでは、OpenAPI定義やその運用について以下のような課題が挙がりました。

- yamlの規模が大きすぎる
- 定義が設計時にしか使われていない
- 記述のルールが定まっていない
- Stoplightを使った運用に課題がある

## 解決方針の検討

課題に対して、いくつかの方法を調査・実験しました。

- rswag
- committee-rails
- TypeSpec
- openapi-typescript
- Stoplight以外のGUIツール
- yaml分割

### 判断基準

- 既存の開発フローに無理なく組み込めるか
- フロントエンドとバックエンドの両方にメリットがあるか
- 導入や移行のコストが大きすぎないか
- 継続して運用できそうか

### 採用したこと・採用しなかったこと

- 採用したこと
  - yaml分割
- 今回は採用しなかったこと
  - rswag
  - committee-rails
  - TypeSpec
  - openapi-typescript
  - Stoplight以外のGUIツール

## 実際にやったこと

### yaml分割

- 分割前の状態
- 分割方針
- ディレクトリ構成
- レビューしやすくするための工夫
